Part 1: Setup Guide

Complete this before the workshop so we can jump straight into exercises. Takes about 15-30 minutes.

What to Bring

  • Laptop - fully charged (power outlets may be limited)
  • Any operating system: Windows, Mac, Linux, or ChromeOS
  • At least 4 GB RAM recommended

Windows Users: Install WSL2 + Ubuntu

If you are on Mac or Linux, skip to the next section. If you are on Windows, WSL2 gives you a proper Linux terminal.

Step 1 Enable WSL and install Ubuntu

Open PowerShell as Administrator (Start menu → type "PowerShell" → right-click → Run as administrator):

wsl --install -d Ubuntu

This downloads and installs Ubuntu. Restart your computer when prompted.

Step 2 Create your Linux username and password

After restarting, Ubuntu will launch and ask you to create a username and password.

  • Username: something short, all lowercase, no spaces (e.g. juan)
  • Password: pick something easy to remember - you will need it when installing software
  • When you type the password, nothing appears on screen - this is normal. Just type and press Enter
Already have WSL? Open PowerShell and type wsl --status. If you see version information, skip ahead to the package managers section.

WSL Survival Skills

How to open your Linux terminal

Click Start → type Ubuntu → click the Ubuntu app. You can also type wsl in PowerShell.

Copy and paste in the terminal

  • Copy: Highlight text with your mouse (auto-copied), or Ctrl+Shift+C
  • Paste: Right-click anywhere in the terminal, or Ctrl+Shift+V

Restart your terminal

Many installers say "restart your terminal." Either close and reopen Ubuntu, or run:

exec bash

Install Git

Git is used for downloading code and version control. You may already have it.

Check if you already have it:

git --version

If you see a version number, you are good. If not, install it below.

Mac:

xcode-select --install

Windows (WSL / Ubuntu):

sudo apt update && sudo apt install git -y

Windows (without WSL):

Download from git-scm.com/download/win and run the installer (accept all defaults).

Linux:

sudo apt install git -y
Git is optional for the workshop. Exercise 3 provides a zip download if you cannot install git.

Install Package Managers

1a Python: Install uv

uv is a fast Python package manager. We use it to run Python projects.

Mac / Linux / WSL:

curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell, if not using WSL):

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Restart your terminal, then install Python and verify:

uv python install 3.12 uv --version

1b JavaScript: Install Node.js

The AI CLI tools are distributed as Node.js packages.

Mac / Linux / WSL (recommended - using Volta):

curl https://get.volta.sh | bash exec bash volta install node

Windows (without WSL):

Download and run the installer from nodejs.org

Verify:

node --version

Install AI Tools

Install at least one. We recommend starting with Gemini CLI since it is free.

2a Gemini CLI (Recommended - Free)

Free with any Google account.

npm install -g @google/gemini-cli

Verify and log in:

gemini --version gemini

When prompted, select "Login with Google." It prints a URL - copy it, paste into your browser, sign in with your Google account, approve, and come back to the terminal.

Alternative: Get a free API key from aistudio.google.com/apikey and set it: export GEMINI_API_KEY="your-key-here"

2b Claude Code (Optional - Subscription Required)

Requires a Claude Pro or Max subscription.

curl -fsSL https://claude.ai/install.sh | bash exec bash

Verify and log in:

claude --version claude

Same browser login flow - it prints a URL, you copy it to your browser, sign in with your Anthropic account, approve, paste the code back.

2c OpenAI Codex (Optional - Subscription Required)

Requires a ChatGPT Plus, Pro, Team, or Edu subscription.

npm install -g @openai/codex

Verify and log in:

codex --version codex

Same browser login flow as the others.

The Authentication Pattern

All three tools follow the same login flow. Once you have done it for one, the rest feel familiar:

Run the tool → It prints a URL → Copy the URL → Paste into browser → Sign in and approve → Get a code → Paste code back in terminal → Done

You only do this once per tool. After that, it remembers you.

Verification Checklist

Run each command below. You should see a version number for each:

  • git --version
  • uv --version
  • node --version
  • gemini --version
  • claude --version (only if installed)
  • codex --version (only if installed)

If any says "command not found", restart your terminal first (exec bash or close and reopen).

Troubleshooting

"Command not found" after installing something

The installer added the tool to your PATH, but your current terminal session does not know about it yet.

Fix: Run exec bash to reload. If that does not work, close the terminal entirely and reopen it.

npm install fails with permission errors

Windows: Run PowerShell as Administrator.

Mac/Linux/WSL: Do NOT use sudo npm install -g. Fix npm permissions instead:

mkdir ~/.npm-global npm config set prefix '~/.npm-global' echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc exec bash

The browser does not open from WSL

This is expected. WSL does not automatically open Windows browsers. When any tool prints a URL:

  • Highlight the URL in the terminal to copy it
  • Open your browser manually (Chrome, Edge, etc.)
  • Paste the URL and press Enter

WSL is running slowly

Make sure you are working with files inside the Linux filesystem. Your Linux home directory (~) is fast. Accessing files on /mnt/c/ (your Windows C: drive) is significantly slower.

Need to update a tool later

npm update -g @google/gemini-cli npm update -g @anthropic-ai/claude-code npm update -g @openai/codex uv self update

Part 2: Workshop Exercises

Open this page on your phone or laptop during the workshop. Click any prompt to copy it.

Exercise 1 App From a Sentence

Describe a problem. Get working software.

Copy this prompt and run it in your terminal:

gemini "Build a sari-sari store POS system. Store owners track inventory, record sales, and see what is selling best. Simple enough for a tindahan owner who uses Facebook but has never touched Excel. Web app with Python."

Run it. Wait 1-3 minutes. Open the app in your browser.

Discussion

  • Would a store owner in Malolos actually use this?
  • What is missing?
  • What did the AI assume that you did not ask for?

Bonus: Change one thing in your prompt and run again. How does the output change?

Exercise 2 Data to Dashboard

Feed data to the CLI. Get analysis and charts back.

Step 1: Download 6 months of sari-sari store sales data (3,200+ rows):

curl -O https://raw.githubusercontent.com/realmindsai/bulsu_tutorial_2026/main/bulacan_sari_sari_sales_6months.csv

Step 2: Analyze it:

gemini "Analyze this 6-month sari-sari store sales dataset. Create: top products chart, daily revenue trend, profit by category, busiest barangays, and weekend vs weekday comparison. Export as an HTML dashboard with peso currency." \ < bulacan_sari_sari_sales_6months.csv

Discussion

  • Open the HTML file in your browser
  • What insights did the AI find that you did not ask for?
  • Which barangay spends the most?
  • What would you add to make it useful for a real store?

Bonus: Ask it to add a "restock alert" section.

Exercise 3 Read a Real GitHub Project

Point the AI at existing code. It understands.

Step 1: Download the heartrate project - a tool that visualizes Python execution in real time (1,800+ stars on GitHub):

curl -O https://raw.githubusercontent.com/realmindsai/bulsu_tutorial_2026/main/heartrate.zip unzip heartrate.zip
Have git installed? You can also use: git clone https://github.com/alexmojaki/heartrate

Step 2: Run the AI inside the folder:

cd heartrate gemini "Read this entire project and generate: 1) A README with what it does and how to use it 2) A list of every file and its purpose 3) How the code works, step by step 4) Any bugs or improvements you can spot Write each to a separate file."

Discussion

  • Did the AI understand what the project does?
  • Could a developer use this documentation to contribute?
  • Did it find real bugs or just style issues?

Bonus: Want to try another project?

git clone https://github.com/danthedeckie/simpleeval

simpleeval: a safe math expression evaluator (570+ stars)

Exercise 4 Build Your Own Specialist

A few sentences turn a general AI into a specialist.

Step 1: Create a system prompt file. Write 3-4 sentences in agent_prompt.md. Ideas:

  • A Bulacan tourism guide for visitors
  • A student budget advisor for tight allowances
  • A recipe assistant for Filipino dishes
  • A study planner for exam season

Step 2: Test it:

gemini --system-instruction agent_prompt.md \ "I have 500 pesos for the week and need to eat 3 meals a day. Make me a plan."

Step 3: Break it. Give your agent a question outside its job.

Ask your tourism guide for medical advice. Ask your budget advisor to write code.

  • Does it stay in role?
  • What rule would you add to make it better?

Exercise 5 Research and Report

One prompt produces a professional document.

gemini "Research the food manufacturing industry in Bulacan province. Produce a briefing: 1) Top companies and what they produce 2) Employment and economic impact 3) Current challenges 4) Three AI-powered opportunities Format as a professional report with executive summary. Export as Markdown and HTML."

Discussion

  • Could you present this to a professor?
  • Could you present it to a local business owner?
  • What would you fact-check first?
  • How long would this have taken manually?

Bonus: Pick a different Bulacan industry and generate another report.

Part 3: Mini Hackathon

6:00 PM - Build something real in 75 minutes

Roles

RoleWhat You Do
PMWrites and refines the prompt
EngineerRuns the AI, iterates on output
TesterTries to break it, finds edge cases
PresenterPrepares the demo

Rotate who talks to the AI. Everyone should try.

Option A: Barangay Service Portal

gemini "Build a web app for barangay complaint tracking. Residents submit complaints with category and location. Officials update status. Include a public dashboard showing resolution rates. Python with SQLite."

Judging: Does it work for a barangay captain?

Option B: Campus Tools for BulSU

gemini "Build a tool for BulSU students that [your idea here]. Think about: class scheduling, study groups, canteen menus, org events, or requirement tracking. Web app with Python backend."

Judging: Would BulSU students actually use this?

Option C: Bulacan Business Intelligence

gemini "Build a CLI tool that generates business reports for Bulacan province. Given an industry name, research key companies, create charts, and export a professional PDF report. Python with matplotlib."

Judging: Could you hand this to an LGU official?

Option D: Your Own Idea

Requirements:

  • Use an agentic CLI tool to build it
  • Must be testable in the demo
  • Must address a local context
gemini "Build [your idea]. [describe users]. [describe core workflow]. Simple enough to demo in 5 minutes."

Hackathon Tips

  • Start small. Get the core working before adding features
  • Iterate. Your first prompt will not be perfect
  • Test outcomes, not code. Does it do what a user needs?
  • Add local knowledge the AI does not have
  • Prepare your demo early
